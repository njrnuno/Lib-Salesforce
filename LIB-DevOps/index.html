DevOps Salesforce everis - Vivo


O(a) profissional de DevOps (abreviação de Desenvolvedor & Operações) é o(a) responsável por otimizar a produtividade dos desenvolvedores e a confiabilidade das operações, desde a etapa de idealização, passando pelo desenvolvimento, até o feedback para garantia de que o produto atenda as necessidades dos clientes. Sua motivação é a comunicação, colaboração e integração contínua entre as duas equipes, promovendo a padronização de processos, liberação e implementação mais veloz de novas versões.

A filosofia pregada pela DevOps é de entrega contínua, com pequenos pedaços de código a cada vez. Considera que as mudanças no contexto de negócio acontecem tão rápido e com tanta frequência que esperar acumular todas elas é colocar em risco o negócio. Os profissionais de DevOps devem preoucupar-se com:

  • entrega contínua de novas funcionalidades, em doses pequenas;
  • uso de equipes dedicadas, cross-functional e pequenas;
  • arquitetura loose coupling;
  • ambiente automatizado por excelência;
  • integração e testes contínuos;
  • ambiente interativo e colaborativo, com usuários, operação e desenvolvimento atuando em conjunto, sem fricções entre os setores.


  • Ambiente de DEV: Ambiente de Desenvolvimento. Destinado para os desenvolvedores atuarem e fazerem as devidas mudanças cabíveis.

  • Ambiente de MERGE: Ambiente entre o Ambiente de Desenvolvilmento e UAT, de modo técnico podemos chamá-lo de Ambiente de Pré homologação. Destinado para uma pré verifição do software, tal deve funcionar de forma espera pelo cliente.

  • Ambiente de UAT: Ambiente de Homologação do cliente. Destinado a fazer exatamente todo processo do ambiente de Merge, validar se o software funciona devidamente como esperado. Os principais objetivos deste ambiente são:
    • Atestar se o software funciona da forma esperada pelo cliente no ambiente proposto;
    • Identificar oportunidades de correção e aplicar os ajustes correlatos à aplicação;
    • Obter o aceite do produto de software pelo cliente;
    • Registrar sugestões para as próximas versões da aplicação.

  • Ambiente de PRODLIKE: O papel deste ambiente é tentar ser o mais próximo da realidade, ou seja, ele deve ser uma réplica perfeita do ambiente de produção e por isso deve-se utilizar as mesmas características do ambiente de produção.

Esteira de Implantação

Segundo o levantamento da State of DevOps de 2018, a maior motivação para adoção de DevOps é necessidade de melhorar qualidade e performance de aplicações. Para cumprir estes desafios, sem perder agilidade no desenvolvimento, é evidente a necessidade de adotar uma esteira de entregas contínuas(pipeline). A imagem acima detalha bem esta esteira empregada no momento.

  • Git for Windows
  • ANT Migration Tools
  • Java JDK
  • Visual Studio Code (Opcional)

Horários das Janelas

Os Horários estão sujeitos a mudanças, mas por determinação parcial durante o decorrer do dia terão três janelas em distintos horários para a Esteira de Implementação. Os horários pré-determinados seguem abaixo:


Segunda Terça Quarta Quinta Sexta
10:30 hrs 10:30 hrs 10:30 hrs 10:30 hrs 10:30 hrs
14:30 hrs 14:30 hrs 14:30 hrs 14:30 hrs 14:30 hrs
16:30 hrs 16:30 hrs 16:30 hrs 16:30 hrs 16:30 hrs

Tabela de Horários Semanais.



Obs: Os pacotes que forem lançados depois das 10:30 hrs só poderão ter seu fluxo iniciado na próxima janela ás 14:30 hrs, os pacotes que não forem lançados até as 14:30 hrs só poderão ter fluxo inicado na próxima janela ás 16:30 hrs. E por fim os pacotes que não forem lançados na Esteira de Implementação até as 16:30 hrs, só poderão ter seu fluxo iniciado no próximo dia respeitando os horários.

  • Merge (União):
